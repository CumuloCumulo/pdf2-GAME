# 测试地点详细分析

## 公共场所

**优势**：
- 低成本 - 安排费用通常较低
- 高流量潜力 - 如果地点选择得当，可吸引大量测试者
- 目标受众访问 - 例如：Schell Games与匹兹堡儿童博物馆合作，获得对3-5岁儿童及父母的持续访问权限

**劣势**：
- 无法筛选特定目标受众 - 任何人都可以参与
- 环境干扰 - 附近的其他活动可能干扰测试者注意力

## 测试者家中

**优势**：
- 自然环境 - 在玩家实际使用的最真实环境中观察游戏
- 社交观察 - 测试者可能邀请朋友，允许观察真实的社会互动

**劣势**：
- 容量有限 - 通常只有1-2名设计师可以参加，每次会话的测试者很少
- 设备负担 - 必须搬运重型设备或花费时间在测试者家中设置控制台环境

## 在线测试

**优势**：
- 参与者池巨大 - 许多人可以参与
- 硬件多样性 - 测试许多不同的计算机配置
- 最适合压力测试和大规模在线游戏玩法问题

**劣势**：
- 反馈质量较低 - 无法获得像面对面测试那样深入、完整的答案
- 保密风险 - 下载游戏可能损害发布前的保密性
- 第一印象风险 - 未完成版本可能损害玩家认知

**缓解措施**：Steam等平台提供安全的beta环境，并带有明确的"未完成游戏"警告来管理期望

## 地点选择决策树

1. 主要目标是什么？
   - 观察深度行为和反应 → 工作室/测试实验室
   - 获得大量快速反馈 → 公共场所/在线
   - 观察真实环境使用 → 测试者家中

2. 预算限制是什么？
   - 高预算 → 测试实验室
   - 中等预算 → 公共场所
   - 低预算 → 在线/工作室

3. 设备要求是什么？
   - 特殊硬件 → 工作室/测试者家中
   - 标准设备 → 公共场所/在线

4. 测试目标群体是什么？
   - 特定年龄段/群体 → 公共场所（特定场所）
   - 一般玩家 → 在线/公共场所
   - 核心玩家 → 工作室/在线
