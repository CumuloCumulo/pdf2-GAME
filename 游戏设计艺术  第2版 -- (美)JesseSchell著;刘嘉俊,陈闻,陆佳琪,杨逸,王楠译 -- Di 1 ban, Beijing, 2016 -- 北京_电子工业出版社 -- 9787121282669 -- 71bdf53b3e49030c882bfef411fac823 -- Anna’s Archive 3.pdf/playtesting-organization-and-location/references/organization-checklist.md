# 试玩测试组织检查清单

## 准备阶段

### 为什么（目标确认）
- [ ] 列出5-10个具体测试问题
- [ ] 确认问题是否可回答
- [ ] 按优先级排序问题

### 谁（测试者招募）
- [ ] 确定需要的测试者类型
- [ ] 制定招募计划
- [ ] 准备筛选问卷（如需要）
- [ ] 安排测试时间表
- [ ] 准备NDA（如需要）

### 在哪里（地点准备）
- [ ] 预订场地
- [ ] 设备准备和测试
- [ ] 环境布置
- [ ] 准备观察设备
- [ ] 准备零食饮料

## 执行阶段

### 测试当天
- [ ] 设备最终检查
- [ ] 欢迎和介绍测试者
- [ ] 说明测试流程
- [ ] 开始测试
- [ ] 观察和记录
- [ ] 收集反馈
- [ ] 感谢测试者

## 后期阶段

### 数据整理
- [ ] 整理观察记录
- [ ] 整理问卷反馈
- [ ] 整理录像/录音
- [ ] 标记关键发现

### 分析和改进
- [ ] 团队复盘会议
- [ ] 确定需要修复的问题
- [ ] 优先级排序
- [ ] 制定改进计划

## 避免的常见错误

1. **目标不明确**："测试游戏是否好玩"过于宽泛
2. **测试者选择不当**：只测试开发者朋友或只测试专家
3. **地点干扰**：工作室环境让测试者不自然
4. **观察不足**：没有观察者或观察者影响测试
5. **引导过强**：给测试者过多提示或指导
