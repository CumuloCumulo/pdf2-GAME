# 开发流程方法论

## 一、迭代法则（绝对真理）

### 核心原则

**「游戏测试和改进的次数越多，就会越出色」**

- 这不是透镜——没有视角
- 这是**绝对真理**，**没有例外**
- 职业生涯中无法绕过

### 危险思维（必须避免）

1. 「这次设计太好了，不需要再测试改进」
2. 「我们别无选择——只能祈祷这是最好的」

每次这样想都会遭受痛苦。

### 电脑游戏的特殊挑战

- 电脑游戏需要大量时间和金钱测试调整
- 系统比传统游戏大得多
- 电脑游戏开发者**别无选择**只能多次迭代
- 这创造了高风险场景

### 两个关键问题

对于测试改进周期长的设计：

1. **怎样才能让每一次迭代都有意义？**
2. **怎样才能尽可能快地进行迭代？**

### 风险评估

游戏开发是赌博：能否在固定预算内让游戏通过所有测试？

- 天真策略：组装游戏并祈祷
  - 有时有效，但失败时→混乱
  - 必须运行可能不够好的游戏，或承受继续开发的痛苦
  - 额外时间和成本常将项目推向亏损

## 二、设计决策哲学

### 快速决策原则

1. **快速做出设计决策**，而非无休止地深思熟虑
2. 一旦内部做出决定（「是的，我要这样做」），缺陷往往在出现前就解决了
3. 利用人类机制：在做决定**之前**考虑不同方面，而非之后

> 「计划就是现实」——斯坦贝克《人鼠之间》

### 承诺与后果分析

- 做出决定后，立即考虑其后果
- 类似抛硬币决策：硬币落地时，你突然知道自己真正想要什么
- 利用人类这一特性

### 愿意推翻决定

- 当意识到错误时，准备推翻之前的决定
- 许多人觉得这困难——一旦做出设计决定，就不愿放弃
- 不要优柔寡断——这是关键心态

### 创意可抛弃心态

| 创意不是 | 创意是 |
|----------|--------|
| 精致瓷器（珍贵、脆弱、不可替代）| 纸杯（廉价、可批量生产）|

如果有人拿走纸杯，再拿一个就是。

这种心态实现更快迭代和更好结果。

### 组合方法

**快速决策 + 果断推翻能力 = 最大化决策能力利用**

游戏设计就是做决定——尽快做出最好的决定。

快速确定想法比拖延更好——你会更快做出好决定，而非花时间考虑潜在选择。

### 反完美主义警告

- 不要沉迷于你的决定
- 当某事不工作时，准备推翻它

## 执行框架

```
快速决策 → 初步承诺 → 分析后果 → 发现错误 → 果断推翻 → 重新决策
     ↑                                                        |
     └────────────────── 迭代循环 ────────────────────────────┘
```

## 检查清单

### 每次迭代
- [ ] 这次迭代是否有明确目标？
- [ ] 是否尽可能快地完成？
- [ ] 是否从测试中学到了东西？

### 每次决策
- [ ] 是否快速做出决定？
- [ ] 是否考虑了后果？
- [ ] 是否准备好推翻决定？
- [ ] 是否对创意保持「纸杯心态」？
