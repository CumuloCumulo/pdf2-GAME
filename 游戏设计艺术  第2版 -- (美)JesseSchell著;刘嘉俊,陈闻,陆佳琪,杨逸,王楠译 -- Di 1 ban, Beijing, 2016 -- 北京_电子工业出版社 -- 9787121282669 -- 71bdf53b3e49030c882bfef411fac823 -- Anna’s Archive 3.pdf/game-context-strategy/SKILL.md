# 游戏情境与平台策略

## 何时使用
- 将真实世界体验转化为游戏
- 选择目标平台
- 根据玩家使用场景设计游戏
- 分析游戏类型与平台的匹配度

---

## 一、本质体验原则

### 核心理念

优秀的游戏不需要完美复制现实，必须捕捉体验的「本质」。

### 提取流程

1. **识别本质**：关键定义特征
2. **识别非本质**：无关细节
3. **转化**：将本质转化为游戏元素

### 案例研究

**打雪仗游戏**：

| 本质 | 非本质 |
|------|--------|
| 雪天停课 | 穿灯芯绒裤子 |
| 适合打包的雪 | 口袋里有薄荷糖 |
| 寒冷但晴朗的蓝天 | 有人遛狗 |
| 到处都是孩子 | |
| 建造堡垒 | |
| 躲避高抛 | |

**转化实现**：
- 视觉：呼吸出的蒸汽
- 音频：呼啸的风声
- 规则：摘下手套改善雪球打包但增加冻伤风险

---

### 成功案例

**Wii Sports棒球**：
- 完全聚焦于挥棒（本质）
- 舍弃9局制、盗垒等细节

**007角色扮演游戏**：
- 引入「英雄点」创造电影般的不可能成功
- 捕捉邦德电影的本质而非现实战争概率

---

## 二、游戏情境分类

### 核心原则

技术（平台）在变，人类情境不变。为情境设计，而非仅为硬件。

---

### 情境1：炉边

**定义**：家的中心（壁炉/电视位置）

**特征**：
- 光线、温暖、安全
- 社交聚集场所
- 进化心理学：盯着火焰会进入恍惚状态

**适用游戏类型**：
- 娱乐群体的游戏（欢迎旁观者）
- 使用身体运动的游戏
- 唱歌、跳舞
- 任天堂Wii类型

**警告**：
- 隔离单个玩家的设计可能失败
- 如WiiU手持焦点在炉边情境中遇到困难

**代表平台/游戏**：
- Nintendo主机（典型客厅焦点）
- 主机社交游戏
- 家庭聚会游戏

---

### 情境2：工作台

**定义**：解决问题的私人角落空间（木工、缝纫）

**特征**：
- 孤独、安静、可能凌乱
- 客人很少造访
- 强独占性

**游戏特征**：
- 难度：高于替代类别
- 紧张度：更激烈
- 时长：每次需要连续数小时

**社交动态**：
- 与其他在线玩家互动
- 分享对游戏挑战重要性的相似观点
- 不与家庭成员（非在线）一起玩

**代表平台/游戏**：
- PC/Steam（Valve策略：统治工作台空间）
- MMORPG（魔兽世界）
- 硬核PC游戏

**未来预测**：
- 头戴式VR设备进入家庭时，将倾向于工作台
- 因为VR体验的孤独性质

---

## 三、平台策略应用

### Steam成功分析

Valve的Steam通过统治工作台空间成功，而竞争对手（索尼、微软、任天堂、苹果）聚焦其他情境。

### 魔兽世界策略

通过针对工作台而非炉边实现更广泛采用。

---

## 四、执行流程

### 设计新游戏时

1. 识别目标体验的本质
2. 确定目标玩家情境（炉边/工作台）
3. 根据情境调整游戏特征：
   - 炉边：降低门槛、增加社交、缩短单局
   - 工作台：增加深度、允许长时间投入
4. 选择匹配的平台

### 分析现有游戏时

1. 识别游戏当前针对的情境
2. 评估游戏特征与情境的匹配度
3. 识别不匹配的问题
4. 提出调整建议
