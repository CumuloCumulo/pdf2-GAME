# 非对称游戏设计

## 何时使用
- 设计非对称游戏玩法
- 平衡玩家技能差异
- 创建个性化游戏体验
- 模拟真实世界的不对等情境

## 前提条件
- 明确的游戏设计目标
- 了解玩家技能分布

## 约束
- 需要更复杂的平衡工作
- 可能让玩家对公平性产生困惑

---

## 五大设计理由

### 理由1：模拟真实世界情境

**逻辑**：真实冲突本质上是非对称的，对称设计会不真实。

**示例**：
- 二战游戏模拟轴心国vs同盟国
- 历史战役重现

---

### 理由2：允许探索游戏空间

**逻辑**：不同角色/能力创造大量不同对局。

**示例**：格斗游戏
- 10个不同角色，每个有独特能力
- 创造10×10 = 100种不同对局
- 每种对局需要不同策略
- 将一个游戏转化为100种不同战略体验

---

### 理由3：实现个性化

**逻辑**：让玩家选择匹配其技能的能力/资源。

**优势**：
- 给玩家权力感和主导权
- 玩家可以发挥自然优势
- 增加玩家投入度

---

### 理由4：平衡技能差异

**逻辑**：用数量或资源补偿能力差距。

**示例**：吃豆人
- 人类大脑在迷宫导航上轻松超越AI
- 4个幽灵vs 1个人类玩家创造公平挑战

**其他应用**：
- 高尔夫让杆系统
- 棋类让子规则

---

### 理由5：创造有趣的情境

**逻辑**：非对称游戏在无限游戏可能性中更常见。

**示例**：尼泊尔Bhag-Chal游戏
- 一方控制5只老虎（目标：吃掉5只山羊）
- 另一方控制20只山羊（目标：困住所有老虎）
- 不同的兵力AND不同的目标
- 新玩家争论公平性，老玩家确认平衡

**好奇心效应**：「哪一方有优势？」的疑问增加趣味

---

## 平衡方法

### 数值平衡法

1. 为每个资源/能力分配数值
2. 确保双方总数值相等
3. 基于实际游戏结果测试和迭代

### 迭代测试流程

1. 初始设计：基于直觉分配资源
2. 内部测试：观察胜率分布
3. 调整数值：微调过强/过弱方
4. 外部测试：扩大测试群体
5. 最终平衡：确认双方胜率接近50%

---

## 决策树

```
IF 需要模拟真实世界情境
   → 考虑非对称设计（理由1）

IF 想增加游戏深度和重玩价值
   → 考虑非对称设计（理由2）

IF 想让玩家表达个性
   → 考虑非对称设计（理由3）

IF 玩家技能差异大
   → 考虑非对称设计（理由4）

IF 想创造独特有趣的情境
   → 考虑非对称设计（理由5）
```
